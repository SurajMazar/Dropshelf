## What this app is

DropShelf is a native macOS AppKit app â€” a floating file-shelf. Read [README.md](README.md) for the user-facing overview and [ARCHITECTURE.md](ARCHITECTURE.md) for the module breakdown before making non-trivial changes.

Key constraints:

- **AppKit, not SwiftUI.** Drag-out to other apps (Finder, browsers, Mail) requires `NSPasteboardWriting` / `NSCollectionViewDelegate` hooks that SwiftUI doesn't expose. Don't port the panel to SwiftUI.
- **Single-window app.** The panel (`ShelfPanel`) has multiple *states* (floating / docked / hidden / with-search) but there's only one window and one `ShelfViewController`. Don't instantiate a second view controller for a "different" mode â€” add state to the existing one.
- **`LSUIElement: true`.** No Dock icon, menu-bar only. Keep it that way.
- **Sandbox is off** for the MVP because shake detection needs a global mouse monitor, which needs Accessibility, which is awkward under sandbox. If you enable sandboxing later, you'll need to either drop shake-to-summon or move it behind an opt-in entitlement.

## Build, run, rebuild

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from [project.yml](project.yml). `.xcodeproj` is in `.gitignore` â€” regenerate it whenever you pull or change `project.yml`.

```bash
# one-time
brew install xcodegen

# every time project.yml changes
xcodegen generate

# build + install locally (Release)
xcodebuild -project DropShelf.xcodeproj -scheme DropShelf -configuration Release \
    CONFIGURATION_BUILD_DIR="$PWD/build/Release" build
pkill -f DropShelf 2>/dev/null; sleep 0.5
rm -rf /Applications/DropShelf.app
cp -R build/Release/DropShelf.app /Applications/DropShelf.app
open /Applications/DropShelf.app

# build a DMG
rm -rf /tmp/DropShelf-dmg && mkdir -p /tmp/DropShelf-dmg
cp -R /Applications/DropShelf.app /tmp/DropShelf-dmg/
ln -s /Applications /tmp/DropShelf-dmg/Applications
hdiutil create -volname "DropShelf" -srcfolder /tmp/DropShelf-dmg \
    -ov -format UDZO DropShelf-0.1.dmg
rm -rf /tmp/DropShelf-dmg
```

## State invariants you must preserve

These invariants exist because of past bugs. Breaking them reintroduces visible glitches.

1. **`setDocked(false)` runs in the undock animation's completion handler, not before.**  
   If you call it up front, the floating UI shows inside the still-tiny window until the expand animation finishes. User reported this as a glitch â€” don't regress.

2. **`setDocked(true)` runs *before* the shrink animation.**  
   The opposite pairing â€” if you reverse it, the user sees the full floating UI briefly crammed into a 48pt window.

3. **`preDockFrame` must be clamped to `visibleFrame` on undock.**  
   Auto-dock saves the window's pre-dock frame, which can be partially off-screen (that's how it got close enough to trigger auto-dock). `clampToVisibleScreen(...)` on undock keeps the expanded window fully visible.

4. **`windowDidMove` for auto-dock is debounced and checks mouse state.**  
   Firing during active drag causes dock/undock churn. Current logic: cancel/re-schedule on each move, only execute after 120ms of stillness AND `NSEvent.pressedMouseButtons == 0`.

5. **`isAnimating` blocks dock / undock / auto-dock from overlapping.**  
   If you add a new animated transition, participate in this flag.

6. **All collection-view index paths map into `store.filteredItems`, not `store.items`.**  
   Any new code reading selection or cell index paths must use `store.filteredItems[idx.item]`. Using `items` directly was a bug â€” stale when search/filter is active.

7. **The entry file must be `main.swift`.**  
   Top-level statements (the NSApp bootstrap) are only allowed in a file literally named `main.swift`. Renaming it reintroduces *"Expressions are not allowed at the top level"*.

## Coding conventions

- No comments unless they explain a *why* that isn't clear from the code (hidden OS quirk, non-obvious ordering, etc.). No "added for X" history comments.
- Don't add abstractions for hypothetical future needs. This is a ~1000-line app; keep it flat.
- Keep the Model layer closure-callback based (`onChange: (() -> Void)?`). Don't pull in Combine or async streams unless you're also removing the closure.
- SF Symbol names â€” if you add one, verify it exists in the SDK you're building against (Xcode 26 / SF Symbols 6). Invalid names silently render as blank buttons.

## Known gotchas

- **Accessibility permission is per-binary-path**. Debug builds live in `DerivedData`, the installed app lives in `/Applications`. These are different paths, so you re-grant on each. This is expected, not a bug.
- **HUD material + `.clear` `backgroundColor` only works if `isOpaque = false`.** Already set in `ShelfPanel`; don't flip it.
- **Drag-out across apps requires non-sandboxed builds** *or* specific entitlements. Current build has sandbox off.
- **The visual effect view must be added as a subview *below* everything else**, or it paints over your content. Current `loadView` ordering is effect â†’ header â†’ separator â†’ empty label â†’ scrollView â†’ searchBar â†’ dockedView.
- **`NSCollectionView` selection index paths are in its own coordinate space**, so using them with `store.items` is wrong when a filter is active. Always go through `store.filteredItems`.
